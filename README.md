# Service Operations Data Demo

This repository demonstrates the type of structured coordination, tracking, and operational data work I supported in my previous roles while collaborating with technical teams.

In those roles, I frequently worked with structured datasets, tracked operational requests, maintained documentation, and helped teams maintain clear overviews of ongoing tasks and issues. This small demo project recreates a simplified version of that workflow using simulated data.

All data in this repository is fictional and created purely for demonstration purposes.

---

## Project Goal

The goal of this project is to demonstrate how service or support operations can be monitored and analyzed using simple data tools.

The workflow represented here includes:

- tracking incoming service requests
- monitoring request status and priority
- identifying high-priority issues
- understanding technician workload
- visualizing operational metrics

---

## Dataset

The dataset represents a simplified after-sales support workflow including fields such as:

- Request ID
- Customer and company
- Product or component involved
- Issue type
- Priority level
- Assigned team member
- Request status
- Follow-up tracking

This reflects the kind of structured tracking often required when coordinating technical requests across teams.

---

## Python Analysis

The Python script performs simple operational analysis using pandas, including:

- total number of service requests
- number of high-priority requests
- requests handled by technicians
- distribution of requests by status

This demonstrates how operational data can be quickly analyzed to support coordination and decision-making.

---

## SQL Queries

The SQL file contains example queries that answer common operational questions:

- How many service requests exist?
- How many requests are in each status?
- Which requests are high priority?
- How many tasks are assigned to each team member?

These queries reflect the type of analysis used when monitoring service workflows.

---

## Power BI Dashboard

A simple Power BI dashboard visualizes key operational metrics:

- requests by status
- requests by priority
- requests by assignee

Dashboards like this help teams quickly understand workload distribution and identify potential bottlenecks.
